
## **O que é DevOps?**

DevOps é uma cultura que combina desenvolvimento (Dev) e operações (Ops), utilizando práticas e ferramentas para acelerar a entrega de software, sem comprometer a estabilidade.

### **Objetivos principais:**

- **Automação** de processos repetitivos.
- **Integração** entre desenvolvimento e operações.
- **Entrega contínua** com ciclos de desenvolvimento mais curtos.
- **Monitoramento e segurança** desde o início do ciclo de vida do software.

## **Funcionamento do DevOps**

- Equipes de desenvolvimento e operações trabalham juntas.
- Automatização de processos de testes, integração e implantação.
- Uso de práticas ágeis e microsserviços.

## **Principais Benefícios**

- **Velocidade:** Entregas mais rápidas e frequentes. 
- **Escalabilidade:** Infraestrutura como código para melhor gerenciamento. 
- **Confiabilidade:** Processos automatizados garantem maior qualidade. 
- **Segurança:** Implementação de DevSecOps para fortalecer a proteção. 
- **Colaboração:** Maior interação entre desenvolvimento e operações.

## **Práticas DevOps**

### **Infraestrutura como Código (IaC)**

Gerenciamento de infraestrutura de forma programática (ex: Terraform, Ansible, AWS CloudFormation).

### **Integração Contínua (CI)**

Automação do envio de código para repositórios e execução de testes (ex: GitHub Actions, Jenkins, GitLab CI/CD).

### **Entrega Contínua (CD)**

Automação do deploy de aplicações (ex: Kubernetes, Docker, ArgoCD).

### **Monitoramento e Logs**

Uso de ferramentas para acompanhar aplicações em tempo real (ex: Prometheus, Grafana, ELK Stack).

### **Segurança e DevSecOps**

A segurança é incorporada desde o início do desenvolvimento (ex: OWASP, Snyk, Trivy).

## **Estágios e Ferramentas DevOps**

1️⃣ **Planejamento** → Jira, Confluence, Trello. 
2️⃣ **Desenvolvimento** → Git, GitHub, GitLab. 
3️⃣ **Build** → Maven, Gradle. 4️⃣ **Testes** → Selenium, JUnit, Cypress. 
5️⃣ **Release/Entrega** → Jenkins, CircleCI. 
6️⃣ **Implantação** → Docker, Kubernetes. 
7️⃣ **Operação** → Ansible, Terraform. 
8️⃣ **Monitoramento** → Splunk, Datadog, Prometheus.

## **DevSecOps: Segurança no DevOps**

- Aplicação de **Shifting Security Left** (segurança desde o início).
- Análise estática e dinâmica de código para identificar vulnerabilidades.
- Monitoramento contínuo de aplicações e infraestrutura.
